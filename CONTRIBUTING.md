# Contributing

Thanks for helping improve Tale of Conquest!

- Branching
  - Create feature branches from `main` (e.g., `feat/<scope>`, `fix/<scope>`)
- Commits
  - Keep messages concise and meaningful; reference issues when possible
- Before opening a PR
  - `npm ci`
  - `npm run lint`
  - `npx tsc -noEmit`
  - `npm run build`
- Pull Requests
  - Keep PRs focused and small when possible
  - Include testing notes and screenshots where useful
  - Request at least one review
